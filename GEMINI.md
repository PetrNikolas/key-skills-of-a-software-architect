# Instructions for Gemini

This repository contains a curated list of key skills and principles for software architects. When working on this project, focus on clarity and brevity. Contributions should be added as items to a flat list in Markdown, formatted exactly as `- 💚 ...` and written in English. The goal is to provide a practical handbook for architects, not to write code.

The full project context (purpose, structure, conventions) is in [PROJECT_CONTEXT.md](PROJECT_CONTEXT.md).
